## 基本概念

### 什么是 LLMClient？

**LLMClient** 是 LLM 模块的工厂类，提供统一的接口来创建和管理 LLM 客户端。它根据提供商类型自动实例化对应的客户端实现。

```python
from src.llm import LLMClient, LLMProvider

# 创建客户端实例
client = LLMClient(
    api_key="your-api-key",
    provider=LLMProvider.OPENAI,
    api_base="http://localhost:8080",
    model="gpt-oss"
)
```

### 什么是 Message？

**Message** 是对话消息的数据模型，包含以下字段：

```python
from src.schema import Message

# 用户消息
user_msg = Message(role="user", content="你好")

# 系统消息（设置 AI 角色）
system_msg = Message(role="system", content="你是一个专业的 Python 程序员")

# 助手消息（包含思考内容）
assistant_msg = Message(
    role="assistant",
    content="这里是回答",
    thinking="这里是思考过程"
)
```

### 什么是 LLMResponse？

**LLMResponse** 是 LLM 响应的数据模型：

```python
response = await client.generate(messages)

# 响应内容
print(response.content)

# 思考内容（如果支持）
if response.thinking:
    print(f"思考过程: {response.thinking}")

# 工具调用（如果触发）
if response.tool_calls:
    for tool_call in response.tool_calls:
        print(f"调用工具: {tool_call.function.name}")
```

| OpenAI | `LLMProvider.OPENAI` | 兼容 OpenAI API，适用于 llama.cpp、本地部署或云端 OpenAI |

## 高级功能

### 工具调用（Function Calling）

工具调用允许 AI 使用外部工具来获取信息或执行操作。

#### 示例 5：使用内置工具

```python
async def tool_calling_example():
    client = await init_client()

    # 定义工具（需要是一个实现特定接口的对象）
    tools = [
        {
            "type": "function",
            "function": {
                "name": "get_weather",
                "description": "获取指定城市的天气信息",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "city": {
                            "type": "string",
                            "description": "城市名称，如：北京、上海"
                        },
                        "unit": {
                            "type": "string",
                            "enum": ["celsius", "fahrenheit"],
                            "description": "温度单位，默认 celsius"
                        }
                    },
                    "required": ["city"]
                }
            }
        }
    ]

    messages = [
        Message(role="user", content="北京今天天气怎么样？")
    ]

    response = await client.generate(messages, tools=tools)

    print("🤖 AI 回复:")
    print(response.content)

    # 检查是否有工具调用
    if response.tool_calls:
        print("\n🔧 工具调用:")
        for tool_call in response.tool_calls:
            print(f"  - 工具名称: {tool_call.function.name}")
            print(f"  - 参数: {tool_call.function.arguments}")

            # 模拟执行工具
            if tool_call.function.name == "get_weather":
                import json
                args = json.loads(tool_call.function.arguments)
                print(f"  - 执行结果: {args['city']} 今天晴天，25°C")
    else:
        print("\n⚠️ 未触发工具调用")

# 运行
# asyncio.run(tool_calling_example())
```

#### 示例 6：自定义工具对象

```python
from typing import Any, Dict
from pydantic import BaseModel

class WeatherTool:
    """天气查询工具"""

    @staticmethod
    def to_openai_schema() -> Dict[str, Any]:
        """转换为 OpenAI 工具格式"""
        return {
            "type": "function",
            "function": {
                "name": "get_weather",
                "description": "获取天气信息",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "city": {"type": "string"}
                    },
                    "required": ["city"]
                }
            }
        }

    def __call__(self, city: str) -> Dict[str, Any]:
        """执行工具逻辑"""
        # 这里可以调用真实的天气 API
        return {
            "city": city,
            "weather": "晴天",
            "temperature": 25
        }

async def custom_tool_example():
    client = await init_client()

    # 使用自定义工具对象
    weather_tool = WeatherTool()
    tools = [weather_tool.to_openai_schema()]

    messages = [
        Message(role="user", content="查询一下上海的天气")
    ]

    response = await client.generate(messages, tools=tools)

    if response.tool_calls:
        for tool_call in response.tool_calls:
            # 执行工具
            import json
            args = json.loads(tool_call.function.arguments)
            result = weather_tool(**args)
            print(f"工具执行结果: {result}")
```

### 思考内容（reasoning_content）

思考内容允许 AI 显示其推理过程，特别适合复杂问题求解和链式思考。

#### 示例 7：启用思考模式

```python
async def thinking_example():
    client = await init_client()

    messages = [
        Message(
            role="user",
            content="请逐步思考并计算: 15 * 23 = ?"
        )
    ]

    response = await client.generate(messages)

    if response.thinking:
        print("🧠 思考过程:")
        print(response.thinking)
        print("\n" + "="*50 + "\n")

    print("💡 最终答案:")
    print(response.content)

    # 保留思考内容到对话历史（重要！）
    messages.append(
        Message(
            role="assistant",
            content=response.content,
            thinking=response.thinking
        )
    )

    # 可以基于之前的思考继续提问
    follow_up = Message(
        role="user",
        content="现在请将结果乘以 2"
    )
    messages.append(follow_up)

    follow_response = await client.generate(messages)

    if follow_response.thinking:
        print("\n🧠 继续思考:")
        print(follow_response.thinking)

    print("\n💡 新答案:")
    print(follow_response.content)

# 运行
# asyncio.run(thinking_example())
```

**输出示例：**
```
🧠 思考过程:
让我逐步计算：
第一步：15 * 23
我可以将 23 分解为 20 + 3
所以 15 * 23 = 15 * (20 + 3)
       = 15 * 20 + 15 * 3
       = 300 + 45
       = 345

==================================================

💡 最终答案:
345
```

### 场景 3：对话助手

```python
class ChatAssistant:
    """简单的对话助手类"""

    def __init__(self, client: LLMClient):
        self.client = client
        self.conversation_history = []

    async def chat(self, user_input: str) -> str:
        """处理用户输入并生成回复"""

        # 添加用户消息
        self.conversation_history.append(
            Message(role="user", content=user_input)
        )

        # 生成回复
        response = await self.client.generate(
            self.conversation_history
        )

        # 添加助手回复
        self.conversation_history.append(
            Message(
                role="assistant",
                content=response.content,
                thinking=response.thinking
            )
        )

        return response.content

    def reset(self):
        """重置对话历史"""
        self.conversation_history = []

async def assistant_example():
    client = await init_client()
    assistant = ChatAssistant(client)

    # 对话
    reply1 = await assistant.chat("你好！我想学习 Python")
    print("助手:", reply1)

    reply2 = await assistant.chat("零基础应该怎么开始？")
    print("\n助手:", reply2)

    reply3 = await assistant.chat("推荐一些学习资源")
    print("\n助手:", reply3)

    print(f"\n📝 对话历史（共 {len(assistant.conversation_history)} 条消息）")

# 运行
# asyncio.run(assistant_example())
```

```python
async def concurrent_requests():
    """并发处理多个请求"""
    client = await init_client()

    # 创建多个对话任务
    tasks = [
        client.generate([
            Message(role="user", content="什么是人工智能？")
        ]),
        client.generate([
            Message(role="user", content="Python 有哪些优势？")
        ]),
        client.generate([
            Message(role="user", content="如何学习机器学习？")
        ])
    ]

    # 等待所有任务完成
    responses = await asyncio.gather(*tasks)

    # 处理结果
    for i, response in enumerate(responses, 1):
        print(f"\n🤖 回答 {i}:")
        print(response.content[:100] + "..." if len(response.content) > 100 else response.content)

# 运行
```

### 场景 6：批量处理

```python
async def batch_processing():
    """批量处理多个问题"""
    client = await init_client()

    questions = [
        "什么是面向对象编程？",
        "解释 Python 中的装饰器",
        "简述 RESTful API 的设计原则",
        "什么是数据库索引？"

    print(f"📦 批量处理 {len(questions)} 个问题...\n")

    results = []

    for i, question in enumerate(questions, 1):
        print(f"[{i}/{len(questions)}] 处理中: {question[:30]}...")

        response = await client.generate([
            Message(role="user", content=question)
        ])

        results.append({
            "question": question,
            "answer": response.content,
            "tokens": len(response.content.split())  # 简单估算
        })

        print(f"  ✅ 完成（{results[-1]['tokens']} 词）")

    # 输出统计
    print(f"\n📊 处理完成:")
    print(f"  总问题数: {len(results)}")
    total_tokens = sum(r['tokens'] for r in results)
    print(f"  总词数: {total_tokens}")
    print(f"  平均词数: {total_tokens / len(results):.1f}")
```

### 4. 错误处理和重试

```python
import asyncio
from asyncio import timeout

async def robust_request(messages, max_retries=3):
    """带重试的请求"""
    client = await init_client()

    for attempt in range(max_retries):
        try:
            response = await client.generate(messages)
            return response
        except Exception as e:
            if attempt == max_retries - 1:
                raise  # 最后一次尝试，抛出异常

            print(f"⚠️ 第 {attempt + 1} 次尝试失败: {e}")
            await asyncio.sleep(2 ** attempt)  # 指数退避

# 使用
```

### 常见问题

#### Q: 如何连接到本地 llama.cpp？

**A:** 确保 llama.cpp 服务已启动，然后使用默认配置：
```python
client = LLMClient(
    api_key="sk-1234567890",
    provider=LLMProvider.OPENAI,
    api_base="http://localhost:8080",
    model="your-model-name"
)
```

#### Q: 模型返回空响应怎么办？

**A:** 检查以下几点：
- API 服务是否正常运行
- 模型名称是否正确
- API 密钥是否有效（如果是云端服务）
- 消息格式是否正确

#### Q: 思考内容为空是正常的吗？

**A:** 思考内容（reasoning_content）是可选功能，取决于模型是否支持。如果模型不支持，响应中的 thinking 字段会为空。

#### Q: 如何提高响应质量？

**A:** 建议：
- 使用系统消息明确 AI 的角色和任务
- 提供足够的上下文信息
- 使用思考模式处理复杂问题
- 保留对话历史维持上下文

#### Q: 支持流式输出吗？

**A:** 当前版本不支持流式输出。响应返回完整的文本内容。流式输出功能已在计划中，将在未来的版本中提供。
