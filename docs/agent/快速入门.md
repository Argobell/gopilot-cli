## 基本概念

### 什么是 Agent？

**Agent** 是智能代理系统，能够自主执行任务、调用工具、管理对话历史。它结合了 LLM 的能力，通过循环执行来完成任务。

```python
from src.agent import Agent

# 创建 Agent 实例
agent = Agent(
    llm_client=llm_client,      # LLM 客户端
    system_prompt="...",        # 系统提示
    tools=[],                   # 工具列表
    max_steps=30,               # 最大执行步数
    workspace_dir="./workspace" # 工作空间目录
)
```

### 关键组件

#### 1. LLM 客户端 (llm_client)

负责与 LLM 通信的客户端实例：

```python
from src.llm import LLMClient, LLMProvider

llm_client = LLMClient(
    api_key="your-api-key",
    provider=LLMProvider.OPENAI,
    api_base="http://localhost:8080",
    model="gpt-oss"
)
```

#### 2. 系统提示 (system_prompt)

定义 AI 角色和行为准则：

```python
system_prompt = """
你是一个专业的编程助手。
你的任务是帮助用户解决编程问题。
请始终提供清晰、准确的代码示例。
"""
```

#### 3. 工具列表 (tools)

可用的工具集合，用于 Function Calling：

```python
tools = [
    # 工具对象列表
    # 每个工具需实现特定的接口
]
```

#### 4. 工作空间 (workspace_dir)

Agent 的工作目录，相对路径会以此为基础：

```python
workspace_dir = "./workspace"  # 工作空间目录
```

#### 5. 消息历史 (messages)

维护对话历史，包含 system、user、assistant、tool 四种类型的消息：

```python
# Agent 会自动管理消息历史
# 包括消息摘要以防止上下文溢出
```

## 基本使用

### 示例 1：初始化 Agent

```python
import asyncio
from src.agent import Agent
from src.llm import LLMClient, LLMProvider

async def init_agent():
    # 创建 LLM 客户端
    llm_client = LLMClient(
        api_key="sk-1234567890",
        provider=LLMProvider.OPENAI,
        api_base="http://localhost:8080",
        model="gpt-oss"
    )

    # 定义系统提示
    system_prompt = "你是一个专业的 Python 编程助手"

    # 创建 Agent
    agent = Agent(
        llm_client=llm_client,
        system_prompt=system_prompt,
        tools=[],  # 暂不使用工具
        max_steps=10,  # 最多执行 10 步
        workspace_dir="./workspace"  # 工作空间目录
    )

    print("✅ Agent 初始化成功！")
    return agent

# 运行
# asyncio.run(init_agent())
```

## 高级功能

### 工具调用（Function Calling）

工具调用允许 Agent 使用外部函数来获取信息或执行操作。

#### 示例 5：使用工具

```python
from src.tools.base import Tool, ToolResult
from typing import Any, Dict
import json

class CalculatorTool(Tool):
    """计算器工具"""

    name = "calculator"
    description = "执行基本数学计算"

    async def execute(self, expression: str) -> ToolResult:
        """执行计算

        Args:
            expression: 数学表达式，如 "15 + 23 * 2"

        Returns:
            计算结果
        """
        try:
            # 安全计算（实际项目中应使用更安全的计算器）
            result = eval(expression)
            return ToolResult(
                success=True,
                content=f"计算结果: {result}"
            )
        except Exception as e:
            return ToolResult(
                success=False,
                content="",
                error=f"计算错误: {str(e)}"
            )

async def tool_calling_example():
    # 创建工具
    calculator = CalculatorTool()
    tools = [calculator]

    agent = Agent(
        llm_client=llm_client,
        system_prompt=(
            "你是一个数学助手，可以使用计算器工具进行计算。"
        ),
        tools=tools,
        max_steps=10
    )

    agent.add_user_message("请计算 15 * 23 + 37")

    result = await agent.run()
    print(result)

# 运行
# asyncio.run(tool_calling_example())
```

#### 示例 6：多工具协作

```python
class FileReaderTool(Tool):
    """文件读取工具"""

    name = "read_file"
    description = "读取文件内容"

    async def execute(self, file_path: str) -> ToolResult:
        """读取文件

        Args:
            file_path: 文件路径

        Returns:
            文件内容
        """
        try:
            with open(file_path, 'r', encoding='utf-8') as f:
                content = f.read()
            return ToolResult(
                success=True,
                content=content
            )
        except Exception as e:
            return ToolResult(
                success=False,
                content="",
                error=f"读取文件失败: {str(e)}"
            )

class DataAnalyzerTool(Tool):
    """数据分析工具"""

    name = "analyze_data"
    description = "分析 CSV 数据"

    async def execute(self, data: str) -> ToolResult:
        """分析数据

        Args:
            data: CSV 格式的数据

        Returns:
            分析结果
        """
        # 简单的数据分析
        lines = data.strip().split('\n')
        return ToolResult(
            success=True,
            content=f"数据行数: {len(lines)}"
        )

async def multi_tool_example():
    tools = [
        FileReaderTool(),
        DataAnalyzerTool()
    ]

    agent = Agent(
        llm_client=llm_client,
        system_prompt="你是一个数据分析师，擅长处理和分析数据",
        tools=tools,
        max_steps=15
    )

    agent.add_user_message(
        "读取 workspace/data.csv 文件并分析数据"
    )

    result = await agent.run()
    print(result)
```

### 消息管理

Agent 自动管理消息历史，并在需要时进行摘要以防止上下文溢出。

#### 示例 8：自定义消息摘要

```python
async def message_management_example():
    agent = Agent(
        llm_client=llm_client,
        system_prompt="你是一个知识问答助手",
        tools=[],
        token_limit=40000,  # 设置较低的 token 限制来触发摘要
        max_steps=20
    )

    # 长时间对话
    questions = [
        "什么是 Python？",
        "Python 有哪些特性？",
        "Python 适用于哪些场景？",
        "如何学习 Python？",
        "Python 的最佳实践是什么？",
        "推荐一些 Python 学习资源"
    ]

    for question in questions:
        agent.add_user_message(question)
        result = await agent.run()
        print(f"\n问题: {question}")
        print(f"回答: {result[:100]}...")
        print("-" * 50)

    # Agent 会自动进行消息摘要
    # 查看消息数量
    print(f"\n当前消息历史数量: {len(agent.messages)}")
```

class TaskAutomationAgent:
    """自动化任务 Agent"""

    def __init__(self, llm_client):
        self.agent = Agent(
            llm_client=llm_client,
            system_prompt="""
            你是一个自动化任务执行助手。
            你可以帮助用户：
            1. 组织和规划任务
            2. 分解复杂任务
            3. 提供执行步骤
            4. 跟踪进度

            请始终提供清晰的任务计划和执行建议。
            """,
            tools=[FileReaderTool()],
            max_steps=30
        )

    async def create_plan(self, task: str) -> str:
        """创建任务计划"""
        self.agent.add_user_message(
            f"请为以下任务创建详细的执行计划：{task}"
        )
        return await self.agent.run()

    async def track_progress(self, task: str, completed_steps: list) -> str:
        """跟踪任务进度"""
        steps_str = ", ".join(completed_steps)
        self.agent.add_user_message(
            f"当前任务: {task}\n"
            f"已完成步骤: {steps_str}\n"
            f"请推荐下一步行动"
        )
        return await self.agent.run()

# 使用示例
async def automation_example():

    # 创建任务计划
    plan = await automation_agent.create_plan(
        "开发一个 Web 应用"
    )
    print("任务计划:", plan)

    # 跟踪进度
    next_step = await automation_agent.track_progress(
        "开发一个 Web 应用",
        ["需求分析", "UI设计", "数据库设计"]
    )
    print("下一步:", next_step)

### 4. 工具设计原则

# ✅ 推荐：清晰的工具定义
class DataAnalyzerTool(Tool):
    name = "analyze_data"
    description = "分析 CSV 数据文件，计算统计指标"

    async def execute(self, file_path: str, metrics: str = "mean") -> ToolResult:
        """
        分析数据文件

### 常见问题

#### Q: Agent 没有触发工具调用怎么办？

**A:** 检查以下几点：
- 工具描述是否清晰明确
- 系统提示是否鼓励使用工具
- LLM 模型是否支持 Function Calling
- 工具参数格式是否正确

#### Q: 如何优化 Agent 的响应速度？

**A:** 建议：
- 减少 `max_steps` 设置
- 降低 `token_limit` 触发更频繁的摘要
- 使用更快的 LLM 模型
- 优化工具执行效率

#### Q: 消息摘要丢失了重要信息怎么办？

**A:** 关键信息应该在：
- 系统提示中明确说明
- 用户消息中重复强调
- 或者提高 `token_limit` 减少摘要频率

#### Q: Agent 进入无限循环怎么办？

**A:** 防止无限循环：
- 设置合理的 `max_steps`
- 在系统提示中明确任务完成条件
- 检查工具是否返回错误状态

#### Q: 如何调试 Agent 的行为？

**A:** 调试方法：
- 查看彩色输出显示的执行步骤
- 打印 `agent.messages` 查看消息历史
- 检查工具调用的参数和结果
- 查看 LLM 响应的 `thinking` 字段
